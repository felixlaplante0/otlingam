# ExactDAG

Exact causal-order learning via subset dynamic programming and Wasserstein
non-Gaussianity.

ExactDAG learns a directed acyclic graph from observational data generated by a
linear structural equation model. It searches globally over causal orders rather
than using a greedy or continuous relaxation. Once an order is found, adaptive
lasso selects edges and estimates their weights.

## Model

ExactDAG assumes a linear structural equation model

$$
X_j = \sum_{i \in \operatorname{pa}(j)} \beta_{ji} X_i + \varepsilon_j,
$$

where the graph is acyclic and the structural noises are mutually independent,
centered, and have finite nonzero variances. The identification result additionally
assumes that at most one structural noise is Gaussian.

Under these assumptions, the causal orders are exactly the maximizers of the
Wasserstein residual objective described below.

## Wasserstein Score

For a candidate order $\pi$, each variable is regressed by ordinary least squares
on all variables preceding it:

$$
R_{\pi_k}(\pi)
= X_{\pi_k}
- \Pi_{\{\pi_1,\ldots,\pi_{k-1}\}} X_{\pi_k}.
$$

Each residual is standardized and compared with a standard Gaussian using the
squared 2-Wasserstein distance:

$$
D(R)
= W_2\left(
    \frac{R}{\sqrt{\mathbb{E}[R^2]}},
    \mathcal{N}(0,1)
  \right)^2.
$$

The score of an order is

$$
G(\pi) = \sum_{k=1}^{d} D\left(R_{\pi_k}(\pi)\right).
$$

For a causal order, the population residuals are the structural noises. A
non-causal order produces at least one residual that mixes multiple standardized
noises. The Wasserstein criterion assigns that mixture a strictly smaller total
score, so maximizing $G$ identifies a topological order of the true DAG.

For finite samples, ExactDAG computes each one-dimensional Wasserstein score by
sorting the standardized residuals and comparing them with standard Gaussian
quantiles.

## Algorithm

Given `n` samples over `d` variables, ExactDAG:

1. Represents subsets of variables with bit masks.
2. Evaluates the local Wasserstein score for each candidate sink and predecessor
   set.
3. Uses subset dynamic programming to select the optimal sink for every subset.
4. Backtracks through the selected sinks to recover a globally optimal order.
5. Applies adaptive lasso along that order to select edges and estimate weights.

The dynamic program evaluates $d2^{d-1}$ local scores and stores $O(2^d)$ states.
Its exponential dependence on `d` is the cost of exact order optimization. Runtime
also depends on the sample size because every local score requires regression,
residual standardization, and sorting.

The computational core is compiled with
[Numba](https://numba.pydata.org/).

```bash
pip install git+https://github.com/felixlaplante0/exactdag.git
```

ExactDAG requires Python 3.11 or later.

Runtime dependencies are NumPy, SciPy, Numba, scikit-learn, and LiNGAM.

## Quickstart

```python
import numpy as np

from exactdag import ExactDAG

rng = np.random.default_rng(42)
n, d = 10_000, 5

# B[j, i] is the direct effect i -> j.
B = np.tril(np.ones((d, d)), k=-1)
noise = rng.uniform(-1, 1, size=(n, d))
X = noise @ np.linalg.inv(np.eye(d) - B).T

model = ExactDAG()
model.fit(X)

print("Causal order:", model.causal_order_)
print("Wasserstein score:", model.score_)
print("Estimated adjacency matrix:")
print(model.adjacency_matrix_.round(2))
```

Set `fit_intercept=False` when the data are already centered. With the default
`fit_intercept=True`, ExactDAG centers the observations and exposes the fitted
intercepts as `model.intercept_`.

## Outputs

After fitting, the estimator provides:

- `causal_order_`: the estimated order from source to sink;
- `adjacency_matrix_`: adaptive-lasso edge weights, with entry `[j, i]`
  representing the effect `i -> j`;
- `score_`: the optimized empirical Wasserstein objective;
- `intercept_`: fitted intercepts when `fit_intercept=True`.
